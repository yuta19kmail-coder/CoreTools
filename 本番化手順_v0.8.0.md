# CoreTools 本番化（CoreFlowログイン＋PitTypeオンラインランキング）手順書 v0.8.0

> CoreTools を GitHub Pages から **Firebase Hosting の独自サブドメイン**（`coretools.kobayashi-motors.com`）へ移し、
> CarFlow/CoreFlow と同じ Google アカウント（CoreFlow名簿＝portalMembers）でログインできるようにする手順。
> コードは実装済み。以下は **ゆうた（＋Firebase Console / DNS）側でやる設定**です。順番どおりに。

---

## なぜサブドメインに移すか（1行）
ログインの戻り先（authDomain）とアプリのURLが**同じドメイン**でないと、iPhone(Safari)でログインが無限ループするため（CarFlowで実証済み）。だから CoreTools 専用のサブドメインを用意します。

---

## ① Firebase Hosting に CoreTools 用サイトを作る（1回だけ）
PowerShellで（`D:\アプリ開発\CoreTools` 内）：
```
firebase login            # 未ログインなら
firebase hosting:sites:create coretools
firebase target:apply hosting coretools coretools
```
- これで carflow-9d500 プロジェクトの中に「coretools」というHostingサイトができ、このフォルダのデプロイ先に紐づきます（`.firebaserc` 設定済み）。

## ② カスタムドメインをつなぐ（DNS）
Firebase Console → **Hosting → coretools サイト → カスタムドメインを追加** → `coretools.kobayashi-motors.com` を入力。
- 表示される **TXT（所有権確認）と A / CNAME レコード**を、ドメイン管理（kobayashi-motors.com のDNS）に追加。
- 反映まで数十分〜数時間。Console が「接続済み（証明書発行済み）」になればOK。

## ③ ログインを許可するドメインを登録（重要）
Firebase Console → **Authentication → Settings → 承認済みドメイン** に
`coretools.kobayashi-motors.com` を **追加**。
- これが無いと「このドメインは未登録です」でログインできません。
- （②のカスタムドメイン接続で `/__/auth/handler` は自動で用意されます）

## ④ Firestore ルールを更新（ランキング保存のため）
PitTypeのスコア用に `pittypeScores` コレクションのルールを追加済み（`CarFlow\carflow\firestore.rules` 内）。
Firebase Console → **Firestore → ルール** に、最新の `firestore.rules` の内容を貼って**公開**。
（または CarFlow フォルダで `firebase deploy --only firestore:rules`）
- 中身：全メンバーが読める／各自は自分のドキュメント(uid)だけ書ける。

## ⑤ デプロイ（公開）
`D:\アプリ開発\CoreTools` で：
```
firebase deploy --only hosting:coretools
```
- 以後の更新もこのコマンド1つ。`https://coretools.kobayashi-motors.com` で開けるようになります。

## ⑥ CoreFlowポータルに「CoreTools＝使える」を反映＋メンバーに付与（誰が入れるかの設定）
CoreToolsは「名簿に居て active、かつ **CoreTools＝使える** がオンの人」だけ入れます（CarFlowと同じ作法。**マスター＝ゆうたは常に入れる**）。
1. ポータル（CoreFlow）を再デプロイ（メンバー編集画面に「CoreTools」トグルを追加済み）：
   - `D:\アプリ開発\ポータル` で `firebase deploy --only hosting`（※ポータルの普段のデプロイ方法に合わせて）。
2. CoreFlow → メンバー管理で、CoreToolsを使わせたい人を選び、**「CoreTools 使える」をオン → 保存**。
   - 既存メンバーは初期 **オフ**（ゆうたが意図的にオンにする運用）。マスターは常にオン。
   - ここをオンにした人だけ、`coretools.kobayashi-motors.com` でログインできる＆ランキングに並べる。

## ⑦ ランチャーのリンク差し替え（任意・後でOK）
各アプリ共通の `js/launcher.js` の coretools URL が今は GitHub Pages
（`https://yuta19kmail-coder.github.io/CoreTools/`）を指しています。
本番化後は `https://coretools.kobayashi-motors.com` に直すと、他アプリの🧰ボタンから新URLへ飛びます。
- launcher.js は全アプリ同一複製なので、差し替えるなら全アプリ分そろえて反映（CarFlow/StockFlow/PitFlow/CoreBoard/CoreNote/CoreTools）。

---

## 動作チェック
1. `https://coretools.kobayashi-motors.com` を開く → ログイン画面が出る。
2. 自分のGoogleアカウントでログイン → 名簿(portalMembers)に居れば入れる。ヘッダーに自分のアバター＋名前。
3. PitType を1回プレイ → 終了時にスコアが保存され、🏆オンラインランキングに自分が並ぶ。
4. 別のメンバーがログイン＆プレイ → 全員が同じランキングに並ぶ（総合金額／スピード／正確さ／スコアベストの4タブ）。

## つまずきやすい点
- **ログインループ／「未登録ドメイン」**：③の承認済みドメイン追加と、②のカスタムドメイン接続が完了しているか。
- **名簿に居ない人が入れない**：CoreFlow（ポータル）の名簿(portalMembers)にそのGoogleメールが登録され、active であること。CoreToolsはアプリ別の利用フラグ(.on)は要求しない＝名簿に居る現役メンバーは全員使える。
- **ランキングが空**：誰か1回プレイすれば載る。保存は練習モードの¥が「総合金額」に積み上がる。
