> 🆕 **2026-06-07（v0.3.0）＝ パーツ税抜計算をベクターPDF化＋「すでに税抜」判定＋列並び替え**
> - **ベクターPDF化**：html2canvas（画像・にじむ）をやめ、**jsPDF＋jspdf-autotable＋日本語フォント埋め込み**に変更＝文字が選択可能なベクターでくっきり。フォントは Noto Sans JP を **jsdelivr から実行時fetch→base64→addFont**（`@expo-google-fonts/noto-sans-jp/400Regular/NotoSansJP_400Regular.ttf`）。初回だけ約5MBのフォント取得（ブラウザキャッシュ）、**jsPDFがサブセット化するので出力PDFは0.1MB前後と軽い**。`partsEnsureFont()`／`_jpFontB64`でセッション内キャッシュ。ライブラリ：pdf.js＋jspdf.umd＋jspdf-autotable（html2canvasは廃止）。
> - **列の並び**：# / パーツNo / 品名 / 数量 / **定価(抜)・仕切(抜)・合計(抜)** / レス率 / **定価(込)・仕切(込)**。抜き3列は太枠（lineWidth0.4）＋仕切(抜)を大きめ(11pt)で強調、込み2列は薄グレー(240)で控えめ。白黒前提・テキストは黒。
> - **「すでに税抜」判定**：請求書PDFの **「内消費税額」が 0 なら税抜版**（同じトーマツでも拠点で税抜/税込が違う：MINI江戸川=税込、江東BMW=税抜）。0のPDFを入れると「すでに税抜の見積書です」と**エラーを出して変換しない**（そのまま10%引きになる事故防止）。0以外＝税込として変換。`partsParse`が`内消費税額`を拾って`d.taxAmt`に。`partsProcess`で`taxAmt===0`なら中断。
> - 検証：実物3枚（WN03241/2E74480=税込→変換OK・数値はゆうたExcelと一致／V778201=内消費税0→エラー）。Nodeでjspdf+autotable+フォント埋め込みのベクター出力も確認（日本語が選択可能テキスト・0.11MB）。

> 🆕 **2026-06-07（v0.2.0）＝ 🧾 パーツ税抜計算ツールを追加**
> - **用途**：トーマツ（MINI/BMW）の請求書PDF（税込）をドラッグ → 税抜計算表PDFを**自動生成＆自動ダウンロード**。受付/事務の日常用＝プレビューなし・手間なし・レスポンス重視。
> - **流れ**：消費税率（10%既定/8%）を選ぶ → 請求書PDFをドロップ（複数可） → 1枚ごとに `税抜計算表_<車台№>.pdf` を即ダウンロード。画面には結果1行だけ（明細数・仕切合計）。
> - **読み取り＝pdf.js**：`cMapUrl`(jsdelivr) ＋ `cMapPacked:true` を必ず指定（この請求書はCID日本語フォントで、CMap無しだと品名が文字化けする＝ハマりポイント）。列は**x座標固定**で判定（行番号x≈50・部品№x≈68・品名x130〜430・数量x≈442・定価x≈464・金額(仕切)x≈518。金額セルは「58,327 25」のように末尾にレス率が付くので半角スペースで切って先頭=仕切）。車台№＝本文の「英数字6〜8桁 + 日付」の最初（例：様 WN03241 2026/06/02）。
> - **計算**：定価(抜)=定価(込)÷(1+税率) 四捨五入／仕切(抜)=仕切(込)÷(1+税率)／合計(抜)=定価(抜)×数量／レス率=仕切÷(定価×数量)−1。**ゆうたの既存Excel（トーマツ定価表）と全数字一致を確認**（WN03241・2E74480の2枚／仕切税込合計=請求書合計と一致）。
> - **出力＝html2pdf**（html2canvasでHTML表をラスタライズ→日本語OK・自動DL）。**仕切(抜)列を強調**（太字・大きめ・薄ハイライト）。小林モータース・取引先名は入れない（ゆうた指定）。
> - 追加ライブラリ：`pdf.js@3.11.174`＋`html2pdf@0.10.1`（cdnjs／CMapはjsdelivr）。コードは `index.html` 内 `renderParts/partsParse/partsMakePdf` 等。
> - ⚠ 既知：請求書フォーマットはこのディーラー専用にx座標を合わせている（決まったフォーマット前提）。別ディーラーが来たら列x座標の調整が要る。
